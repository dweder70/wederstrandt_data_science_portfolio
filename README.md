# Data Science Portfolio

This repository contains ten portfolio projects that demonstrate practical skills in data preparation, exploratory analysis, statistical modeling, machine learning, forecasting, and technical communication.

## Portfolio Highlights

- End-to-end analytical workflows from data acquisition through communication.
- Mixed tooling across Python, Jupyter, R, and R Markdown.
- Project artifacts in notebook, report, and presentation formats.
- Domain coverage across healthcare, finance, energy, weather, education, and public policy.

## Naming and Documentation Conventions

This repository uses domain-focused names where possible, such as "Fraud Detection" or "Time Series Forecasting," instead of course or program identifiers.

Recommended conventions for future additions:

- Prefer names that describe domain + method.
- Use lowercase snake_case for script and notebook file names.
- Keep folder-level README files focused on objective, data, methods, outputs, and reproducibility.

## Reproducibility Baseline

1. Use Python 3.10+ for notebook and script-based projects.
2. Use a recent R installation for R Markdown projects.
3. Run notebooks from top to bottom to preserve execution order.
4. Review each folder README for project-specific setup notes.

## Notes

- Some projects are documentation-first and primarily include report/presentation outputs.
- Some projects include rendered outputs in addition to source files for reviewer convenience.
